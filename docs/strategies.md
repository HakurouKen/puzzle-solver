# Star Battle 求解策略

> 本文件聚焦"求解器用了哪些推理策略、为什么有效、何时触发"，不涉及实现细节。
> 实现见 `solve-k.ts`（通用骨架 + 通用策略）、`solve-2.ts`（k=2 特化）、`solve.ts`（k=1 特化）。

---

## 1. 问题定义

在 `n × n` 棋盘上放置星，使得：

1. 每一行恰 `k` 颗星；
2. 每一列恰 `k` 颗星；
3. 每一个区域恰 `k` 颗星；
4. 任意两颗星在 King 邻接（含 8 个方向）下不相邻。

输入：`regions[i][j]` 给出格 `(i,j)` 所属的区域 id。

本仓库分三层组织：

| 文件          | 角色                                                                 |
|---------------|----------------------------------------------------------------------|
| `solve-k.ts`  | 通用骨架：类型 / 辅助 / 6 个 k 无关的通用策略 / deduce / search / `solveWith(regions, k, strategies)` |
| `solve-2.ts`  | k=2 特化：复用 solve-k 通用部分 + 注入 `regionShapeEnum`、`forcedChain`(k=2)  |
| `solve.ts`    | k=1 特化：复用 solve-k 通用部分 + 注入 `forcedChain`(k=1)、`hiddenRowGroup`、`hiddenColGroup`  |

`solve-k.ts` 自身也直接可用：`solve(regions, k=2)` 仅用通用策略集（无形状枚举 / 强制链 / 隐藏对偶），适合任意 k 但效率较弱。

---

## 2. 策略总览

求解过程是**推理 fixpoint + 有限回溯搜索**：每轮按固定顺序尝试下列策略，命中即写回状态；全部静默时再分支搜索。

| 优先级 | 策略                | 类型    | 一句话说明                                            | 通用 | k=1 | k=2 |
|--------|---------------------|---------|-------------------------------------------------------|------|-----|-----|
| 1      | uniqueRegion        | 定星    | 区域候选数 = 剩余需放数（need）→ 全部定星               | ✓    | ✓   | ✓   |
| 2      | uniqueRow           | 定星    | 行候选数 = need → 全部定星                              | ✓    | ✓   | ✓   |
| 3      | uniqueCol           | 定星    | 列候选数 = need → 全部定星                              | ✓    | ✓   | ✓   |
| 4      | saturationClear     | 划 X    | 星 8 邻 / 已满线 / 已满区域的剩余格全划 X              | ✓    | —*  | ✓   |
| 5      | regionShapeEnum     | 定星/X  | 区域候选数小（≤6）时直接枚举所有合法放法，取交集/补集 |      | —** | ✓   |
| 6      | rowColRegionLock    | 划 X    | 未放星区域候选全在同一行/列 → 该线他区格全划 X         | ✓    | —*  | ✓   |
| 7      | generalizedPair     | 划 X    | k 个未放星区域候选行/列并集恰 k 条 → 这些线他区格划 X  | ✓    | ✓   | ✓   |
| 7'     | hiddenLineGroup     | 划 X    | sz 行/列候选区域并集恰 sz 个 → 这些区域线外格划 X       |      | ✓   | —** |
| 8      | forcedChain         | 定星/X  | 试放某候选 → 致他区无解 → 该候选划 X（剩 1 个则定星）  |      | ✓   | ✓   |

> *— k=1 下隐含于"已用行/列/区"判定中，但 saturationClear / rowColRegionLock 在 k=1 下仍 sound 可用，solve.ts 中保留。
> **— 在另一 k 下退化为更弱的已有策略，未启用。
> "通用"列：✓ 表示策略写在 `solve-k.ts` 的 `GENERIC_STRATEGIES` 中，任意 k 求解器可直接复用。


---

## 3. 策略详解

### 3.1 唯一候选三连：uniqueRegion / uniqueRow / uniqueCol

**思路**：约束推到极限——某个"必须放 k 颗星的容器"（区域 / 行 / 列）剩余需放数 `need` 与候选数 `cells.length` **恰好相等**时，这些候选必为星，全部定下。`need = k - 已放数`，所以 `need=1` 时退化为"只剩一个候选 → 定星"。

**为什么有效**：最便宜的剪枝，O(候选数)。一旦某线收紧到候选 = 需放数，立即变为已知星，其他策略借此再收紧。

**k 适用性**：通用，写在 `solve-k.ts` 的 `GENERIC_STRATEGIES` 中。k=1 时 `need` 始终为 1（or 0），退化为最简形式；k≥2 时能多颗一起定。

---

### 3.2 saturationClear：饱和清除

**思路**：把"显然不可能放星"的格子打上 X：

- 任一已知星的 8 邻格；
- 已满 `k` 颗星的行、列、区域内的剩余格。

**为什么有效**：把"不可能"显式记录下来，后续候选派生跳过它们，避免每个策略重复检查相邻。

**k 适用性**：通用，写在 `solve-k.ts`。k=1 下"已满 k 颗 = 已用过 1 次"已经被 deriveCandidates 过滤覆盖，但策略仍 sound，作为统一入口保留。k=2 下因为同一行/列/区可放 2 次但占满后剩余仍要划 X，必须独立成策略才能覆盖。

---

### 3.3 regionShapeEnum：区域形状枚举（仅 k=2）

**思路**：对候选数 ≤ 6 的区域，**直接枚举所有"互不 King 相邻"的合法放法**：

- 在所有放法中**都包含**的格 → 这格必是星（取交集）；
- 任一放法都**不包含**的格 → 这格不可能是星（取补集，划 X）。

**经典样例**：

- **1×3 直线区域，need=2**：互不相邻的 2 颗只能落在两端 → 端点定星，中点不定。
- **2×3 矩形区域，need=2**：枚举可见 2 颗永远不在中列 → 中列两格补集划 X。

**为什么有效**：把"区域形状 + 互不相邻"这两个约束合在一起做局部全枚举，能挖出广义对、强制链都看不到的几何结论。

**k 适用性**：k=1 下 `need=1`，"枚举 1 颗放法"等价于 `uniqueRegion`，无新增推力。

---

### 3.4 rowColRegionLock：行列-区域互锁

**思路**：若某未放星区域的全部候选都落在**同一条线**（行或列）上，那条线的 `k` 颗星必由该区域贡献 → 该线上其他区域的格不可能放星，全划 X。

**保解前置**：必须 `regionCount[rid] === 0`（区域尚未放任何星）。否则区域只需再放 `k - regionCount` 颗（< k），无法填满整条线，他区也可合法占用 → 不能锁定。

**为什么有效**：把"区域 ⊆ 一条线"的几何巧合放大成对整条线的排他约束。

**k 适用性**：通用，写在 `solve-k.ts`。k=1 下与 `generalizedPair sz=1` 等价，但前者更便宜（O(区域数)），保留。

---

### 3.5 generalizedPair：广义对（区域 → 线）

**思路**：找 `sz` 个未放星区域，使得它们的候选**行**（或**列**）并集恰好为 `sz` 条线 → 这 `sz` 条线的 `k·sz` 颗星必由这 `sz` 个区域贡献 → 这些线上其他区域的格全划 X。

`sz` 从 2 起枚举（`sz=1` 即 rowColRegionLock，独立处理更便宜）。

**保解前置**：参与 combo 的区域必须 `regionCount === 0`。已放星区域只需再放 `k - regionCount < k` 颗，无法独占整条线，combo 不再是排他闭包。

**为什么有效**：是数独中"X-Wing / Swordfish"的推广。把局部"区域候选受限"信息聚合成跨多线的硬约束。

**k 适用性**：通用，写在 `solve-k.ts`。任意 k 都 sound。

---

### 3.6 hiddenLineGroup：隐藏对偶（线 → 区域，仅 k=1）

**思路**：generalizedPair 的对偶视角。找 `sz` 行（或 `sz` 列），使得这 `sz` 行的候选**区域**并集恰好为 `sz` 个区域 → 这 `sz` 个区域必把它们的星放在这 `sz` 行内，把它们在**这些行外**的候选全部划掉。

**为什么独立有效**：在 k=1 下"sz 行 ↔ sz 区域"是天然的一一对应（每行每区都恰 1 星）。从"哪些区域**只能**贡献给这些行"的角度看，能挖到 generalizedPair 看不到的剪枝。在数独中对应"Naked Pair / Hidden Pair"的对偶关系，二者不等价、可独立推进。

**k 适用性**：k=1 时是 `solve.ts` 在原 6 步策略基础上**新增**的两条（行/列两套）。k=2 下"sz 行的星可能由 sz+ 个区域贡献"，对偶不再天然成立，未实现。

---

### 3.7 forcedChain：强制链（试验消除）

**思路**：对候选数 2~3 的区域，**逐候选试放**：若假设在 `(r,c)` 放星会让某个其他区域**无任何合法候选**（即所有候选都与 `(r,c)` 冲突），则 `(r,c)` 是"致死候选"，划 X。如果只剩 1 个非致死候选，直接定星。

**关键判定**：他区候选 `(or, oc)` 与试放 `(r, c)` "不冲突" 等价于"非 King 相邻"——这是 k=2 下保解修复的要点（修复 1）。

- **k=1 下**：同行/同列已天然不能共存（每行每列只 1 颗星），可附加 `or !== r && oc !== c`。
- **k=2 下**：同行不相邻、同列不相邻可共存（行/列还能再放 1 颗），**只能**用 King 相邻判定。

**为什么有效**：是仅次于完全枚举的最强单步策略。能解决"两条候选互相牵制，但只有一条会引发跨区域死锁"的局面，常常在其他策略全部静默时一击解局。

**k 适用性**：k=1、k=2 都适用，判定细节不同。

---

## 4. 调度：从策略到求解

策略不是孤立运行的，而是构成一个**双层结构**：

### 4.1 推理 fixpoint（deduce）

按上表优先级**逐一尝试**策略：命中 → 写回状态 → 重新派生候选 → 回到第 1 个策略再来一遍。直到一整轮所有策略都静默才停止。

这样安排的考量：

- **顺序即优先级**：便宜的（uniqueRegion）在前，昂贵的（forcedChain，需对每候选试放）在后。一旦便宜策略产出，立刻重启循环让它和后续策略都用上新信息。
- **保解前置**：rowColRegionLock / generalizedPair 都要求 `regionCount === 0`，否则会把合法候选误划 X。
- **顺序由测试守住**：`STRATEGIES` 数组的顺序被一条专门的测试锁定，重构时不会被静默打乱。

### 4.2 回溯搜索（search）

推理跑到不动点仍未解完时，进入回溯：

- 选**候选数最少的区域**作为分支点（MRV，最小剩余值启发式）；
- 对该区域每个候选 `(r,c)`：克隆当前状态（保解 / 分支隔离）→ 在副本里假设放星 → 递归求解；
- 失败回退到下一个候选。

**回溯预算**：在区域=行的退化盘（`rowsBoard(8)`）上实测 ~272 次回退，策略集合从基线版的 ~2639 削减约 90%。

## 5. 保解性约束（k=2）

策略集合从初版到当前版的关键修复围绕**保解性**——不能因为一条策略的过度推断把合法解排除掉。最终审查发现并修复了 3 处过排：

1. **forcedChain 的他区无解判定**：早期附加 `or !== r && oc !== c`（要求他区候选与试放点既不同行也不同列），这在 k=2 下错把"同行不相邻共存"的合法局面判为冲突。修复：仅保留 King 相邻判定。
2. **rowColRegionLock 缺饱和前置**：区域已放部分星时仍锁定整条线 → 误排他区。修复：加 `regionCount[rid] === 0` 前置。
3. **generalizedPair 缺饱和前置**：combo 含已放星区域时，combo 不再独占整组线，排他不成立。修复：rids 过滤 `regionCount[rid] === 0`。

每条修复都对应一条**精确回归测试**：逐一回退该处修复，对应测试必然变红，其他测试保持绿色——这是终审采纳的回归测试硬标准。

## 6. 求解器对照

| 维度                   | `solve-k.ts` (任意 k)               | `solve-2.ts` (k=2)                          | `solve.ts` (k=1)                              |
|------------------------|-------------------------------------|---------------------------------------------|-----------------------------------------------|
| 入口签名               | `solve(regions, k=2)`               | `solve(regions)`（k 固定 2）                | `solve(regions)`（k 固定 1）                  |
| 通用骨架与策略         | 自身实现（被两 k 特化版复用）       | 复用 `solveWith`、6 个通用策略              | 复用 `solveWith`、6 个通用策略                |
| 形状枚举               | —                                   | regionShapeEnum                             | —                                             |
| 隐藏对偶（线 → 区域）  | —                                   | —                                           | hiddenRowGroup / hiddenColGroup               |
| 强制链                 | —                                   | forcedChain（仅 King 相邻判定）             | forcedChain（含 `or!==r && oc!==c`）           |

通用策略集（`GENERIC_STRATEGIES` in `solve-k.ts`）顺序：uniqueRegion → uniqueRow → uniqueCol → saturationClear → rowColRegionLock → generalizedPair。两个 k 特化版按各自策略类型插入：

- **k=2 顺序**：uniqueRegion → uniqueRow → uniqueCol → saturationClear → **regionShapeEnum** → rowColRegionLock → generalizedPair → **forcedChain**
- **k=1 顺序**：uniqueRegion → uniqueRow → uniqueCol → saturationClear → rowColRegionLock → generalizedPair → **hiddenRowGroup → hiddenColGroup → forcedChain**

顺序均由各自测试中的"策略顺序守卫"锁定，重构不会被静默打乱。
